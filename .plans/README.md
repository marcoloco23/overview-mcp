# .plans/

Short design notes, one per non-trivial feature. **Write a plan before implementing any
new module or external surface** (a new client, a new tool group, a new dashboard card
type). Plans survive context compaction and let a fresh agent pick up mid-feature.

How:
1. Copy `_TEMPLATE.md` → `YYYY-MM-DD_<slug>.md`.
2. Fill Goal / Approach (with the option you chose) / Implementation steps / Files / Testing.
3. Update the plan's Status as you go; mark COMPLETED when the Definition of Done is met.

Not needed for: small fixes, tweaks, or following a step already spelled out in ROADMAP.md.
