# Plan: Phase 2 — Active fires (NASA FIRMS)

**Date**: 2026-06-05
**Status**: COMPLETED (live FIRMS call deferred pending a free FIRMS_MAP_KEY)
**Phase**: ROADMAP Phase 2

## Goal

Add a `fires_in` tool that returns active-fire detections from NASA FIRMS for a bbox, and
render them as markers on the dashboard map. First tool that needs a (free) API key.

## Background

ROADMAP Phase 2. FIRMS area API returns near-real-time thermal-anomaly/fire detections as
CSV. `FireDetection` type already exists in `src/types.ts`. Endpoint:
`GET https://firms.modaps.eosdis.nasa.gov/api/area/csv/{MAP_KEY}/{SOURCE}/{west,south,east,north}/{dayRange}[/{date}]`

CSV columns differ by sensor: VIIRS uses `bright_ti4`, MODIS uses `brightness`; confidence
is a string (low/nominal/high) for VIIRS and 0–100 for MODIS. Parse by header name so one
parser handles both. FIRMS returns plain-text errors (e.g. "Invalid MAP_KEY") instead of
CSV — detect by absence of a `latitude,longitude` header and surface a clean error.

## Approach

- Add `fires()` to `src/clients/nasa.ts` (keep all NASA endpoints in one client) + a
  header-keyed `parseFiresCsv()`.
- New `src/tools/fires.ts` registering `fires_in`; require `FIRMS_MAP_KEY` with a clear
  message + link if missing (graceful, not a stack trace).
- Dashboard: `showFires()` in `web/src/map.ts` (orange flame-dot markers, popup with
  confidence/FRP/date, cap markers), a `fires` branch in `cards.ts`, and `focusCard` in
  `main.ts`.

## Testing

- Parser unit-check against a representative VIIRS CSV sample (no key needed).
- Push a synthetic `fires` card → screenshot dashboard → confirm markers render.
- MCP `listTools` shows `fires_in`; calling it with no `FIRMS_MAP_KEY` returns a clean
  "set FIRMS_MAP_KEY" error (not a crash).
- If a real `FIRMS_MAP_KEY` is available: live call over a current fire region.

## Risks / edge cases

- Sensor-specific columns (VIIRS vs MODIS) → header-keyed parsing.
- Hundreds of detections → cap markers (~800) and note truncation.
- FIRMS area-size / day-range / transaction limits → fail gracefully on non-CSV body.
